# Rep5x - Versioning

Version tracking for Rep5x hardware components using semantic versioning.

## Version format: `vMAJOR.MINOR.PATCH`

- **MAJOR** (`v1.0.0` → `v2.0.0`): Breaking changes, new parts required
- **MINOR** (`v1.0.0` → `v1.1.0`): Compatible improvements, optional upgrades
- **PATCH** (`v1.0.0` → `v1.0.1`): Minor fixes, direct replacements

## Current version: v1.1.0

### Compatibility:
- All v1.0.0 parts work with v1.1.0 parts
- Mix and match freely within v1.x versions
- Only print new parts if you want the improvements

## File naming:
3D printed parts include version: `B_arm_v1.1.0.3mf`
